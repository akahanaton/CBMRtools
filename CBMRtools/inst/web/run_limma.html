wrapper for running limma with additional columns attached

Usage

run_limma(eset, treatment, cond, control, verbose = TRUE, sort.by = "adj.P.Val", decreasing = FALSE, cutoff = NA)

Arguments

eset
expression set object with gene expression
treatment
column name in pData(eset) specifying the conditions for differential expression
cond
value in treatment specifying the positive condition
control
value in treatment specifying the control condition
verbose
whether to display additional columns in report table
sort.by
column to sort by
decreasing
direction for sorting
cutoff
threshold value for keeping results

Description

wrapper for running limma with additional columns attached

Examples

data(eSet.brca.100)
eset<-eSet.brca.100
eset<-eset[, eset$ER_status %in% c("Positive", "Negative")]
res<-run_limma(eset = eset, treatment = "ER_status", cond = "Positive", 
	control = "Negative", verbose = TRUE, sort.by = "adj.P.Val", decreasing = FALSE, cutoff = 0.05)
